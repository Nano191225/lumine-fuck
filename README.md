🛡️ Windows firewall for **Minecraft Bedrock Edition multiplayer** that detects and blocks malicious IPs using reverse DNS, ASN lookup, and direct IP/CIDR rules. **Windows only.**

> 日本語版: [README.ja.md](README.ja.md)

## Features

### Detection
- **Reverse DNS Filtering** — Resolves remote IPs via PTR query using Google/Cloudflare DNS (8.8.8.8, 1.1.1.1) and blocks IPs whose hostname ends with a configured suffix (e.g. `.orangevps.com`)
- **IP / CIDR Blocking** — Directly block specific IPs or subnets without needing rDNS (e.g. `1.2.3.4`, `10.0.0.0/24`)
- **Minecraft Process Filtering** — Only inspects traffic from Minecraft-related processes (`Minecraft.Windows`, `javaw`, `java`), ignoring all other applications
- **UDP + TCP Monitoring** — Captures UDP traffic via [Npcap](https://npcap.com/) / SharpPcap and polls the TCP table via `GetExtendedTcpTable` P/Invoke

### Blocking
- **Windows Firewall Integration** — Creates native Windows Firewall rules (TCP/UDP × Inbound/Outbound) via `HNetCfg.FwPolicy2` COM API. No kernel driver required.
- **Block Delay** — Configurable delay (default: 5s) between detection and actual blocking, allowing brief connections to complete normally
- **Auto-Unblock** — Automatically removes firewall rules after a configurable time (default: 10s). IP is re-detected and re-blocked on next connection attempt
- **Clear on Stop** — All firewall rules are removed when protection is disabled

### GUI
- Dark-themed WPF interface (Catppuccin Mocha)
- Live **Blocked Connections** table and **Activity Log**
- Real-time statistics: blocked IPs, scanned IPs, rule count
- **Config** window: manage domain suffixes, IP/CIDR rules, and all settings
- In-app auto-update via Velopack

## How It Works

```
New TCP/UDP connection (Minecraft process only)
        ↓
1. IP/CIDR match?         → YES → [Block Delay] → Block
        ↓ NO
2. rDNS PTR lookup        (Google/Cloudflare DNS)
   Domain suffix match?  → YES → [Block Delay] → Block
        ↓ NO
   Ignored

After auto-unblock timeout:
  IP removed from firewall + detection cache → re-detectable on next connection
```

## Installation

### Requirements
- Windows 10/11 x64
- [Npcap](https://npcap.com/#download) — required for UDP monitoring (installer will prompt if missing)
- Administrator privileges (required for Windows Firewall access; the app self-elevates via UAC)

## Configuration

Click **⚙️ Config** in the main window to open the settings panel.

### Blocked Domain Suffixes
IPs whose rDNS hostname ends with a suffix are blocked.

Default: `.orangevps.com`

Persisted to `%AppData%\LumineFuck\blocklist.json`.

### Blocked IP Addresses / CIDR
Block specific IPs or subnets directly — no rDNS needed.

Examples: `1.2.3.4`, `20.202.59.0/24`

### Advanced Settings

| Setting | Default | Description |
|---------|---------|-------------|
| Show notifications when blocked | On | Show a desktop notification each time a connection is blocked |
| Auto-unblock after (seconds) | 10 | Remove firewall rule N seconds after blocking. 0 = never unblock |
| Block delay (seconds) | 5 | Wait N seconds after detection before blocking. 0 = block immediately |
